FAERS processed — README
=========================

Overview
--------
This folder contains processed FAERS artifacts derived from the quarterly FAERS ASCII dumps in the sibling `raw` folder. Processed outputs are intended for downstream analysis (EDA, modeling, linking to Orange Book) and to provide compact working tables.

Structure
---------
- `code/` — processing and reporting scripts (see descriptions below).
- `*.csv` — final merged datasets (ANDA-linked FAERS, samples).
- `figures/` — EDA figures generated by the EDA scripts.
- `out/` — inventory CSVs describing the raw ASCII files (size/rows by quarter & table).
- filtered subsets — drug-filtered outputs (CSV and parquet) are saved in this folder when generated (see Key files below).

Key files (scripts)
-------------------
- `code/2026-05-12-faers_ob_all_drugs.py` — Merged FAERS/ORANGE BOOK linking script (dated). Discovers available `raw/faers_ascii_{YYYY}q{Q}` folders, reads `DEMO`, `DRUG`, `OUTC` tables per quarter, filters to Primary Suspect (`PS`) drug rows, links to Orange Book by `Appl_No`, computes severity flags and period/year variables, and writes a compact ANDA-linked CSV to `processed/` with an automatically derived filename `faers_all_drugs_anda_linked_{start_y}q{start_q}_{end_y}q{end_q}.csv`.

- `code/2026-05-12-faers_ob_eda.py` — EDA script (dated). Loads the latest `faers_all_drugs_anda_linked_*.csv` and produces a small set of diagnostic plots (rows by year, severity distribution) into `processed/figures/`.

- `code/2026-05-12-faers_ascii_inventory.py` — Inventory wrapper (dated). Runs the inventory routine (from `2026-01-09-faers_ascii_inventory_all_drugs.py`) to produce CSV summaries describing the raw ASCII file sizes and record counts. Outputs are placed in `processed/out/` as:
  - `faers_ascii_file_inventory_all_drugs.csv`
  - `faers_ascii_totals_by_quarter_all_drugs.csv`
  - `faers_ascii_totals_by_table_all_drugs.csv`

- `code/2026-05-12-faers_valisure_filter_and_eda.py` — Valisure filter + EDA (dated). Loads the latest merged ANDA-linked FAERS CSV, uses the Valisure FEI/name sheet (`FEIs_March 2026.xlsx`) to identify the 14 Valisure drugs, saves the filtered dataset to `processed/` as `faers_valisure_14_drugs_2026-05-12.csv` and (when parquet dependencies are available) `faers_valisure_14_drugs_2026-05-12.parquet`, and writes diagnostic plots to `processed/figures/` (rows by year, severity distribution). Matching is performed by name/FEI token matching and is approximate; mapping by NDC/FEI is recommended for deterministic joins.

- `code/2026-01-09-faers_ascii_inventory_all_drugs.py` — Original, full inventory script. Scans `raw` ASCII folders, counts rows and sizes for each `.txt` table and summarizes by quarter and table.

- `code/2026-01-06-eda_faers_ob_metformin_severity.py` — Legacy EDA focused on metformin severity outcomes. Contains additional diagnostic plots specific to metformin and was used earlier in exploration. May be consolidated into the dated EDA script if desired.

- Older files with different naming conventions may exist; if you want me to remove or archive them, say which ones to keep.

Raw data note (short)
---------------------
- Raw FAERS quarterly ASCII folders are stored under `Data/15 - FDA - Adverse Event/raw/` with names like `faers_ascii_2015q1`, `faers_ascii_2015q2`, etc. Each quarter folder contains multiple `$`-delimited `.txt` tables (DEMO, DRUG, OUTC, etc.).
- Inventory scripts read those folders to compute sizes and row counts without loading full tables into memory.
- The merged script discovers available quarter folders automatically; adding new `faers_ascii_{YYYY}q{Q}` folders in `raw/` will cause the merge script to include them on the next run.
